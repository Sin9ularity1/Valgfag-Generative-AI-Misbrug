About the Project

This elective project concerns the misuse of generative artificial intelligence (GenAI) within IT security. The project investigates how modern AI systems can be exploited in attacks such as prompt injection, AI-supported phishing, and social engineering.

The purpose is to analyze concrete misuse scenarios and identify technical countermeasures with a focus on practical cybersecurity.

Technical Glossary

To help navigate the technical aspects of this project, this table defines the core concepts used throughout the research and analysis.

Term Definition
GenAI (Generative AI) Artificial intelligence capable of generating new content (text, images, code) rather than just analyzing existing data.
LLM (Large Language Model) A type of AI (like GPT-4) trained on massive datasets to understand and generate human-like text based on probability.
RAG (Retrieval-Augmented Generation) An architecture that allows an LLM to "look up" information from a specific database before generating an answer, improving accuracy.
Prompt Injection A security exploit where a user provides crafted input that tricks the AI into ignoring its safety rules or original instructions.
Vector Embedding A mathematical representation of text where words/sentences are converted into numbers so the computer can calculate "semantic similarity."
Agency The level of autonomy given to an AI, such as the ability to delete files, send emails, or access a database without human approval.
System Prompt The "hidden" set of instructions given to the AI by developers that define its personality, rules, and safety boundaries.
Token / Tokenization The process of breaking down text into smaller units (tokens) that the model can process. This is the "alphabet" of an LLM.
Hallucination When an AI confidently generates information that is factually incorrect or completely fabricated.
Attack Surface All the different points where an attacker could try to enter data into or extract data from a system.
Data/Model Poisoning When an attacker corrupts the data used to train the AI, introducing hidden backdoors or biases into the model's logic.
PII (Personally Identifiable Information) Sensitive data that can identify an individual, such as names, social security numbers, or private emails.

Motivation

As a student of IT security, I wanted to supplement my previous focus on traditional security principles with a newer threat area. Generative AI introduces new attack surfaces that are already actively exploited, making the topic relevant both academically and practically.

Focus Areas and Technologies

  • Analysis of AI-generated phishing
  • Social engineering using AI
  • AI-assisted malware
  • Prompt injection and manipulation
  • Deepfakes and identity misuse
  • Automated reconnaissance
  • Python and optionally local AI model for demonstration

Project Goals

  • Investigate how generative AI can be used in cyberattacks
  • Identify central threat vectors related to AI
  • Analyze concrete examples of AI-based misuse
  • Develop a simple local AI model as a demonstration tool (optional)
  • Conduct an in-depth analysis of one attack type
  • Discuss defensive strategies and countermeasures

Method

The project is carried out as a combination of literature study, technical analysis, and selected practical demonstrations. The focus is on understanding concrete attack techniques and evaluating defensive strategies.

  • Mapping of GenAI-related attack methods
  • Analysis based on OWASP Top 10 for LLMs
  • Illustrative scenarios and limited hands-on tests
  • Ongoing documentation via logbook
