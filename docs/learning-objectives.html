Learning Objectives

The following objectives define the targeted knowledge, skills, and competencies for this elective project.

1. Knowledge Objectives

Upon completion, the project will demonstrate an understanding of:

  • The threat landscape surrounding generative AI and Large Language Models
  • Key attack techniques such as prompt injection, AI-supported phishing, and social engineering
  • OWASP Top 10 for LLMs and related security frameworks
  • Typical vulnerabilities in GenAI-based systems
  • Basic technical countermeasures against AI-related attacks

2. Skill Objectives

The student will demonstrate the ability to:

  • Analyze and document GenAI-based attack scenarios
  • Identify security vulnerabilities through literature study and illustrative examples
  • Structure technical documentation on a web-based platform
  • Communicate complex security issues clearly and systematically
  • Apply existing security frameworks for practical risk assessment

3. Competency Objectives

The student will be able to independently:

  • Critically evaluate generative AI from a cybersecurity perspective
  • Plan and execute an independent security project
  • Reflect on defensive strategies and risk reduction
  • Translate theoretical knowledge into practical security analysis
  • Communicate technical security aspects to both technical and non-technical audiences

4. Reflections

Throughout this elective project, I have successfully bridged the gap between theoretical security frameworks and the emerging challenges of Generative AI. The transition from studying the OWASP Top 10 for LLMs to performing a structured Vulnerability Analysis has significantly deepened my understanding of how traditional security principles apply to non-deterministic systems.

Key reflections on the learning process include:

  • Evolution of Threats: Realizing that while many AI vulnerabilities mirror traditional web attacks (e.g., Prompt Injection vs. SQL Injection), the "reasoning-based" nature of LLMs requires a fundamental shift in defensive thinking.
  • Practical Application: Developing the technical demonstration and analysis has moved my skills from passive knowledge to active security evaluation, fulfilling the core competency of translating theory into practice.
  • Strategic Communication: Structuring this research as a technical documentation site has reinforced the importance of communicating complex, evolving risks to a broader audience—a critical skill for any security professional.

This project has not only met the defined objectives but has also provided a robust foundation for ongoing specialization in AI security and defensive strategies.
