# Appendix 1: Technical Glossary

This glossary provides simple, easy-to-understand definitions for the technical terms used in this project.

| Term | Noob-Friendly Definition |
| :--- | :--- |
| **GenAI (Generative AI)** | AI that **creates** things (like writing a story or making a picture) instead of just sorting data. |
| **LLM (Large Language Model)** | The "brain" of the AI (like ChatGPT). It has read almost everything on the internet so it can talk just like a human. |
| **RAG (Retrieval-Augmented Generation)** | Like giving an AI an **open book** to look at while it answers you, so it doesn't have to rely only on its memory. |
| **Prompt Injection** | Tricking the AI into breaking its own rules by using clever wording. (Like a "Simon Says" game where the hacker wins). |
| **Vector Embedding** | Turning words into numbers so a computer can "measure" how similar they are. For example, "Cat" and "Kitten" are mathematically very close. |
| **Agency** | How much **freedom** the AI has to do things on its own, like deleting files or sending emails without asking for permission first. |
| **System Prompt** | The "secret" background rules the AI has to follow. It’s what tells the AI to be helpful and not to be mean or dangerous. |
| **Token / Tokenization** | Chunks of words. The AI doesn't see the word "Apple" as letters; it sees a specific code for that chunk of text. |
| **Hallucination** | When the AI lies to you but sounds **extremely confident** about it. |
| **Attack Surface** | All the possible "doors" or "windows" an attacker could use to try and break into the system. |
| **Data/Model Poisoning** | Sneaking "bad" info into the AI's lessons while it's learning, so it grows up with hidden backdoors or biases. |
| **PII (Personally Identifiable Information)** | Anything that can identify you personally—like your address, phone number, or private emails. |
